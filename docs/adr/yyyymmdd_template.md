<!--
ファイル名のフォーマット: {yyyymmdd}_{title}.md
yyyymmdd: ADRを作成した日付
title: ADRのタイトルを簡潔に表す単語をスネークケースで記述
例: 20250117_エラーイベントはUiModelに持たせる.md
-->

<!--
ステータス
- 提案中: PRを作成した状態
- 採用: PRがマージされた状態
- 廃止: 別のADRによって決定を覆す場合はファイル名を{yyyymmdd}_廃止_{title}.mdに変更する
  - 廃止されたADRはそのまま残し、廃止されたことを明記する
  - ファイルの先頭に「[{yyyymmdd}_{title}](/{yyyymmdd}_{title}.md)によって廃止されました」と記載する
-->

# コンテキスト
<!--
ADR作成に至った背景や経緯
読み手が決定を理解することを助けます
-->

# 決定
<!--
意思決定そのものについて書きます
肯定的かつ命令的な文章で、読み手を考慮した簡潔な言葉を用いて書きます
-->

# 影響
<!--
この決定によるメリット、デメリット、他の選択肢を選ばなかった理由
詳細に書くことで「どのような影響を想定していたか」を明確にし、読み手が意思決定の妥当性や必要性を理解することを助けます
-->

# コンプライアンス
<!--
決定が守られるための手段を書きます
手段としてレビューを挙げることが多いです
コンプライアンスを書くことで、成果物が決定に基づいて作られることを助け、チームや組織における責任の所在が明確になります
-->
