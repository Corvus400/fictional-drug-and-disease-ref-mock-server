# ADR: README に陳腐化する情報を書かない

## ステータス

採用

## コンテキスト

README.md が 329 行に膨張し、以下の問題が発生していた:

- 手動セットアップ手順（JDK インストールコマンド、Apple Container インストーラー URL）がバージョンアップのたびに陳腐化する
- Apple Container CLI リファレンスは公式ドキュメントの劣化コピーである
- Android エミュレーター接続方法は Android側 側の設定に依存し、Mock Server の README で管理すべき情報ではない
- 既知の問題（Activity Monitor 文字化け）は Mock Server 固有の問題ではなく、macOS のバグである
- Admin API の全エンドポイント一覧はコードから自動生成される Swagger UI / カタログと重複する

## 決定

README.md には以下の基準を満たす情報のみ記載する:

1. **プロジェクトの概要**（何であるか）
2. **クイックスタート**（最短で動かす手順）
3. **アーキテクチャの要約**（設計判断の根拠）
4. **開発コマンド**（ビルド・テスト・lint）
5. **自動生成ドキュメントへのポインタ**（カタログ、Swagger UI、ReDoc）

以下は README から除外する:

- 手動セットアップ手順 → `scripts/setup.sh` が自動化済み
- Apple Container CLI リファレンス → 公式ドキュメント参照
- Android エミュレーター接続方法 → Android側 リポジトリの責務
- 既知の問題 → Issue トラッカーの責務
- Admin API 全エンドポイント一覧 → Swagger UI / カタログが自動生成
- 詳細な curl 使用例 → Admin API の基本パターンのみ残す

## 結果

- README.md を約 120 行に圧縮
- コードから導出可能な情報は自動生成ドキュメント（カタログ、Swagger UI、ReDoc）に委譲
- 陳腐化リスクのある情報を排除し、メンテナンスコストを削減
